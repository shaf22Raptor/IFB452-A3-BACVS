Manage Existing Credentials

Revoke cancels the credential. Replace creates a new active credential using the student reference, document hash, and credential type above, then marks the current Credential ID as superseded.

Who uses this tab: the student wallet that holds a credential. Use the Credential Context panel above to set which credential you are acting on, then grant or revoke verifier access here.

Manage Verifier Access

Decide which verifier wallets are allowed to read your credential.

My Credentials

List every credential currently held by the connected wallet.

Who uses this tab: a verifier wallet (e.g. an employer) that the holder has granted access to. All actions operate on the Credential ID set in the Credential Context panel above.

Verify Credential Validity

Confirms the credential exists and is currently active. Every verification is automatically recorded on-chain as an auditable log entry.

Compare Document Hash

Confirms a document you were given matches the hash stored on-chain. The comparison is always recorded on-chain. Set the Document Hash in the Credential Context panel before running.
